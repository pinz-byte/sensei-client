# M1 — Author Station

**Role:** Authors and maintains `sensei_client`, HERALD adoption, AVT adoption, contract, advisories. Ships to `github.com/pinz-byte/sensei-client`. Consumes SENSEI for its own integration testing.

**Network:** Must reach M3 on port 8000.

## Bootstrap

```bash
cd "/sessions/cool-dazzling-edison/mnt/SENSEI M1"   # or wherever your clone lives
./setup/bootstrap.sh M1 --m3-url http://<M3_HOST>:8000
```

What it does on M1 specifically:

1. Verifies Python ≥ 3.9
2. Creates `venv/` in the repo root
3. Installs `sensei_client` **editable from the local tree** (`pip install -e .`) — changes to `sensei_client/*.py` take effect immediately, no reinstall needed
4. Writes `.env` with `SENSEI_STATION=M1`, the M3 URL, and `SENSEI_ADAPTER_SPEC` pointing at the in-tree HERALD spec
5. Runs the smoke test against M3

## Daily workflow on M1

**Editing the library:**

```bash
source venv/bin/activate
set -a; source .env; set +a
# edit sensei_client/*.py — changes take effect instantly (editable install)
python tests/e2e_smoke.py   # verify no regression
```

**Authoring a new adapter:**

```bash
mkdir -p sensei_adapters/<venture>
# Author <venture>.v1.json following sensei_adapters/herald/herald.v1.json
# Author <venture>_materiality.py following the HERALD or AVT pattern
# Write a README.md documenting the four factors
# Validate with a smoke run before declaring done
```

**Cutting a new version:**

```bash
# 1. Bump version in pyproject.toml and sensei_client/__init__.py
# 2. Commit: git add -A && git commit -m "Bump to vX.Y.Z"
# 3. Tag: git tag vX.Y.Z
# 4. Push: git push origin main && git push origin vX.Y.Z
# 5. Optional: build wheel — python -m build -o dist/
```

## Common issues

- **`.env` overwritten on re-bootstrap.** Expected. The bootstrap is idempotent; any env customizations you've added manually should go in a separate file (e.g. `.env.local`) and be sourced after `.env`.
- **M1 pushed a new tag but consumers still pull the old one.** Consumers cache git refs. Either bump the version number or have them `pip install --force-reinstall`.
- **Smoke test fails scenario 4 (fail-open).** That scenario points at `http://127.0.0.1:1` deliberately; if it fails, your firewall or docker network may be intercepting unreachable-address resolution. Not a blocker — scenarios 1–3 cover the production path.
